    **1. shared/project-filter.mjs:**

    `filterProjects(pattern, projectNames)`:
    - Split on comma first: `pattern.split(',').map(s => s.trim()).filter(Boolean)`
    - For each segment:
      - If it contains `*`: convert to regex by escaping special chars and replacing `*` with `.*`, test against each project name
      - Else: exact match against project names
    - Return deduplicated array of matching project names
    - This is a simple wildcard matcher (only `*` supported) -- per RESEARCH.md "Don't Hand-Roll" section, `*` is the only wildcard needed for project names

    **2. deps-command.mjs:**
    Entry-point script. Parses process.argv for arguments.

    **Argument parsing:**
    - First positional arg = project name (required)
    - `--reverse` flag = show reverse dependencies
    - `--depth N` flag = limit tree depth (default: Infinity for full depth)
    - If no positional arg: print error via output-format and `process.exit(1)`

    **Main logic:**
    - Get workspace root: `process.env.CLAUDE_PROJECT_DIR || process.cwd()`
    - Call `loadIndex(workspaceRoot)`
    - Validate project exists in index. If not: print `error("Project '" + name + "' not found (" + meta.projectCount + " indexed)")` and suggest `nx show projects`. Exit 1.
    - Build and print the dependency tree

    **Tree building (forward deps):**
    - Start from the named project
    - Use a recursive function `printTree(name, depth, visited, firstOccurrences)`:
      - Track visited nodes in a Set for circular detection
      - Track firstOccurrences in a Map to detect deduplication
      - Root node: print just the name (no "- " prefix)
      - Children: print with "  " * level + "- " prefix
      - If node was already visited in current path (circular): print with "!" suffix, stop recursing, increment circular count
      - If node was already expanded (first occurrence elsewhere): print with "^" suffix, stop recursing, increment dedup count
      - If `--depth N` reached: stop recursing (don't print children beyond depth)
    - After tree: print legend line `^ = deduped, ! = circular`
    - Print summary: `N nodes (X direct, Y unique, Z deduped, W circular)` where:
      - N = total node appearances in tree
      - X = number of direct dependencies of root
      - Y = unique project names in tree
      - Z = deduped count (^ markers)
      - W = circular count (! markers)

    **Tree building (reverse deps with --reverse):**
    - Invert the dependency graph: for each project, find all projects that depend on it
    - Build a reverse adjacency list from `index.dependencies`
    - Use the same tree-printing logic with the reversed graph

    **Output format per user decision:**
    ```
    my-app
      - shared-utils
      - feature-auth
        - shared-utils ^

    ^ = deduped, ! = circular
    4 nodes (2 direct, 3 unique, 1 deduped, 0 circular)
    ```

    **find-command.mjs:**
    Entry-point script. Parses process.argv for arguments.

    **Argument parsing:**
    - First positional arg = search pattern (required)
    - `--project <name|glob>` flag = scope to specific projects (optional)
    - `--context N` flag = context lines (optional, default: 0)
    - If no positional arg: print error and exit 1

    **Main logic:**
    - Get workspace root
    - Call `loadIndex(workspaceRoot)`
    - If `--project` specified: use `filterProjects` to resolve project names. If no matches, print error with project count and exit 1.
    - Resolve source roots from matched projects (use `sourceRoot` if present, fall back to `root`)
    - Detect regex mode: if pattern starts with `/` and ends with `/`, strip delimiters and use regex mode
    - Call git grep using the "Git Grep Scoped Search Pattern" from RESEARCH.md:
      - `spawnSync('git', args, { cwd: workspaceRoot, encoding: 'utf8', windowsHide: true, maxBuffer: 5MB })`
      - Args: `['grep', '-n', '--no-color']` + `-F` for fixed string (omit for regex) + `-C${context}` if context > 0 + `'--'` + pattern + sourceRoots
      - Exit code 1 = no matches (not an error), exit code > 1 = actual error
    - Parse output: each line is `file:lineNum:content`
    - Group results by project: for each match, determine which project's sourceRoot contains the file path
    - If unscoped (no --project) and matches > 20: truncate to 20 and print `warn("Showing 20 of " + total + " matches. Use --project to narrow results.")`

    **Output format per user decision:**
    ```
    my-app (3 matches)
      apps/my-app/src/main.ts:15: import { something } from '@myorg/shared-utils';
      apps/my-app/src/app.ts:8: import { auth } from '@myorg/feature-auth';
      apps/my-app/src/app.ts:22: const result = auth.validate(token);

    shared-utils (1 match)
      libs/shared-utils/src/index.ts:3: export function something() {

    4 matches in 2 projects
    ```

    Export a `runFind(pattern, index, options, workspaceRoot)` function for testability that returns `{ output, exitCode }`.
