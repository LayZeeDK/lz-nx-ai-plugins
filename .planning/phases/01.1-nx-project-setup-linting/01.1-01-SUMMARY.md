# Phase 01.1 Plan 01: Nx Project Setup and Linting Summary

Nx project discovery, typecheck targets, ESLint flat config with module boundaries and import ordering, test conversion from .mjs to .ts with full type safety.

## What Was Done

### Task 1: Create project infrastructure and config files

- Installed `@nx/eslint-plugin`, `typescript-eslint`, `eslint-config-prettier`, `eslint-plugin-import-x`
- Created `plugins/lz-nx.rlm/project.json` with explicit `typecheck` target (`tsc -p tsconfig.json --noEmit`) and `type:plugin` tag
- Created `plugins/lz-nx.rlm/tsconfig.json` with `allowJs: true`, `checkJs: true` for JSDoc type checking of `.mjs` files
- Created `tests/lz-nx.rlm/tsconfig.json` with `#rlm/*` path alias, vitest globals types, and `allowJs: true` for resolving plugin `.mjs` imports
- Created root `eslint.config.mjs` with Nx flat/base, flat/typescript, flat/javascript presets, module boundary enforcement (type:plugin and type:test constraints), and `import-x/order` for import ordering
- Added `#rlm/*` path alias to `tsconfig.base.json` with relative `./` prefix for `moduleResolution: nodenext` compatibility
- Added `type:plugin` tag to `lz-nx-rlm` and `type:test` tag to `lz-nx-rlm-test`
- Updated `vitest.config.mjs` include pattern from `**/*.test.mjs` to `**/*.test.ts`
- Added `format` and `format:check` scripts to `package.json`
- Ran `nx format --all` to normalize formatting across the workspace

### Task 2: Convert tests to TypeScript and fix all type/lint errors

- Renamed all 8 test files from `.test.mjs` to `.test.ts` using `git mv`
- Added JSDoc `@typedef` definitions for `WorkspaceIndex`, `GraphOutput`, `DepsMap`, `ProjectEntry`, `Dep`, `TargetConfig`, `GraphNode`, `GraphDep`, `AliasIndex`, `ExecError` across plugin scripts
- Replaced all `@param {object}` annotations with specific `Record<string, T>` or named typedef types
- Added type assertions for `catch (err)` blocks (`/** @type {Error} */` or `/** @type {ExecError} */`)
- Added explicit type annotations for dynamically imported test functions using `any` parameter types where the test passes `undefined` to verify error handling
- Added `NodeError` interface and `ExecError` interface in test files for `Error.code` and `Error.stdout`/`Error.stderr` properties
- Removed unused imports (`afterEach`, `vi`, `tsconfigFixture`, `mockBuildIndex`) and unused destructured variables
- Created `tests/lz-nx.rlm/package.json` with `"type": "module"` for ESM support
- Fixed `import-x/order` config: replaced `newlines-between: 'never'` with `'ignore'` due to eslint-plugin-import-x v4.16.1 false-positive bug

### Task 3: Full verification suite

- `nx run-many -t typecheck`: Clean (0 errors) for both projects
- `nx run-many -t lint`: Clean (0 errors, 0 warnings) for both projects

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Fixed paths in tsconfig.base.json require relative prefix**
- **Found during:** Task 1
- **Issue:** `moduleResolution: "nodenext"` requires paths values to start with `./` when no `baseUrl` is set
- **Fix:** Changed `"#rlm/*": ["plugins/lz-nx.rlm/scripts/*"]` to `"#rlm/*": ["./plugins/lz-nx.rlm/scripts/*"]`
- **Files modified:** `tsconfig.base.json`
- **Commit:** b2e6355

**2. [Rule 1 - Bug] Fixed eslint-plugin-import-x false-positive on newlines-between**
- **Found during:** Task 2
- **Issue:** `eslint-plugin-import-x` v4.16.1 incorrectly reports "There should be no empty line between import groups" when `node:*` builtin import is followed by `vitest` external import with no blank line between them and `newlines-between: 'never'`
- **Fix:** Changed `newlines-between` from `'never'` to `'ignore'` to still enforce import ordering without the buggy newline detection
- **Files modified:** `eslint.config.mjs`
- **Commit:** 0b9ac29

**3. [Rule 3 - Blocking] Added package.json type:module for ESM support in tests**
- **Found during:** Task 2
- **Issue:** TypeScript `moduleResolution: nodenext` treats `.ts` files as CJS when the enclosing `package.json` lacks `"type": "module"`, causing `import.meta` to error with TS1470
- **Fix:** Created `tests/lz-nx.rlm/package.json` with `{ "type": "module" }` to mark test directory as ESM
- **Files modified:** `tests/lz-nx.rlm/package.json` (created)
- **Commit:** 0b9ac29

**4. [Rule 1 - Bug] Removed unused import of `warn` from find-command.mjs**
- **Found during:** Task 2
- **Issue:** `warn` was imported from `./shared/output-format.mjs` but never used in `find-command.mjs`
- **Fix:** Removed unused import
- **Files modified:** `plugins/lz-nx.rlm/scripts/find-command.mjs`
- **Commit:** 0b9ac29
