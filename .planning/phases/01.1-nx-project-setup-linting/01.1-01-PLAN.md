    ```json
    {
      "name": "lz-nx-rlm",
      "tags": ["type:plugin"],
      "targets": {
        "typecheck": {
          "command": "tsc -p {projectRoot}/tsconfig.json --noEmit",
          "cache": true,
          "inputs": [
            "{projectRoot}/scripts/**/*.mjs",

          ],
          "outputs": []
        }
      }
    }
    ```

    The explicit typecheck target override is REQUIRED because `@nx/js/typescript` silently disables inferred typecheck when `noEmit: true` is in tsconfig (see RESEARCH.md Pitfall 1). The command uses `tsc -p` (not `tsc --build`) because `composite: false` in the project tsconfig makes `--build` mode invalid.

    **Step 5: Create `plugins/lz-nx.rlm/tsconfig.json`**

    ```json
    {
      "extends": "../../tsconfig.base.json",
      "compilerOptions": {
        "allowJs": true,
        "checkJs": true,
        "noEmit": true,
        "composite": false,
        "declaration": false,
        "declarationMap": false,
        "emitDeclarationOnly": false,
        "rootDir": "scripts"
      },
      "include": ["scripts/**/*.mjs"],
      "exclude": ["node_modules"]
    }
    ```

    Critical: `composite: false` overrides base to avoid TS5053 error. `declaration: false`, `declarationMap: false`, `emitDeclarationOnly: false` override base to prevent output. `allowJs: true` + `checkJs: true` enables JSDoc type checking on .mjs files.
