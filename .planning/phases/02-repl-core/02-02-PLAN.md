    - Simple code: "print('hello')" -> { output: "hello", variables: {}, final: null, finalVar: null, error: null }
    - Variable persistence: "const x = 42; print(x)" -> { output: "42", variables: { x: 42 }, final: null, ... }
    - FINAL call: "FINAL('answer is 42')" -> { final: "answer is 42", ... }
    - FINAL_VAR call: "FINAL_VAR('results')" -> { finalVar: "results", ... }
    - Error handling: "throw new Error('boom')" -> { error: "boom", ... }
    - Timeout: infinite loop with short timeout -> { error: contains "timed out", ... }
    - Security: "eval('1+1')" -> { error: contains "Code generation from strings", ... }
    - Session restore: variables from session state accessible in code
    - Session write: new variables written to session file after execution
    - Object.prototype.toString patched: "print('val: ' + {a:1})" -> output contains JSON, not "[object Object]"
    - Code transformation: "const x = 42" persists x on sandbox (would vanish without transform)
    - SHOW_VARS() works inside sandbox code
    - Config-driven timeout: when options.timeout absent, loadConfig(pluginRoot, workspaceRoot) provides maxTimeout (seconds * 1000 = ms)
    - Config-driven timeout fallback: when pluginRoot also absent, uses DEFAULTS.maxTimeout * 1000

    CLI entry point (stdin -> stdout):
    - echo 'print("hello")' | node repl-sandbox.mjs --index path --session path [--timeout 30000] [--plugin-root path] [--workspace-root path]
    - When --timeout not provided, calls loadConfig(pluginRoot, workspaceRoot) and uses config.maxTimeout * 1000
    - Writes single JSON line to stdout (SandboxResult)
    - Reads code from stdin using readFileSync(0, 'utf8') (cross-platform, per CLAUDE.md)
    - Parses --session, --index, --timeout, --plugin-root, --workspace-root from process.argv

  </behavior>
  <action>
    Create repl-sandbox.mjs with two parts:

    PART 1: Exported `executeSandbox(code, options)` function (testable without process):

    1. Determine timeout: if `options.timeout` is provided (number), use it directly. Otherwise, call `loadConfig(options.pluginRoot || null, options.workspaceRoot || null)` and use `config.maxTimeout * 1000` (seconds to ms). This ensures the guardrails config's maxTimeout value from lz-nx.rlm.config.json is honored as the default, not a hardcoded fallback. If loadConfig cannot find any config files (both paths null), DEFAULTS.maxTimeout (120s = 120000ms) is used.
    2. Read session state: `readSession(options.sessionPath)` (or {} if no sessionPath)
    3. Load workspace index: `JSON.parse(readFileSync(options.indexPath, 'utf8'))` (or {} if no indexPath). NOTE: This intentionally uses direct JSON.parse, NOT loadIndex(). The sandbox is a per-invocation process that receives a pre-built index file path. The caller (Phase 3 agent) is responsible for calling loadIndex with auto-rebuild logic before spawning the sandbox. Direct file read avoids the overhead of loadIndex's freshness checks on every REPL turn.
    4. Create print capture: `createPrintCapture(2000, 20000)`
    5. Create final handlers: internal object with FINAL(answer) and FINAL_VAR(name) setters + getters
    6. Create REPL globals: `createReplGlobals(index, options.workspaceRoot, printCapture, finalHandlers)`
    7. Create SHOW_VARS closure over the sandbox object (needs reference to sandbox after creation)
    8. Build sandbox object: `{ ...sessionState, ...globals, SHOW_VARS, console: { log: print, error: print, warn: print } }`
    9. Create VM context: `vm.createContext(sandbox, { codeGeneration: { strings: false, wasm: false } })`
    10. Run Object.prototype.toString patch in the context (Hampton-io pattern):
       ```javascript
       vm.runInContext(`Object.prototype.toString = function() { try { return JSON.stringify(this, null, 2); } catch(e) { return '[object Object]'; } };`, ctx)
       ```
    11. Transform code: `transformDeclarations(code)` (prepends strict mode)
    12. Execute: `vm.runInContext(transformedCode, ctx, { timeout: resolvedTimeout })` where resolvedTimeout is from step 1
    13. Catch errors: set error field in result
    14. Write session state: `writeSession(options.sessionPath, sandbox, BUILTIN_GLOBAL_NAMES)` (if sessionPath provided)
    15. Return SandboxResult: `{ output: printCapture.getOutput(), variables: extractState(sandbox), final: finalHandlers.getFinalAnswer(), finalVar: finalHandlers.getFinalVarName(), error }`

    The extractState function filters sandbox entries: skip BUILTIN_GLOBAL_NAMES, skip functions, skip non-JSON-serializable values. This provides the `variables` field in SandboxResult for informational purposes (separate from session file write).

    PART 2: CLI entry point (at bottom, guarded by process.argv check):
    1. Read code from stdin: `readFileSync(0, 'utf8')` (fd 0, cross-platform)
    2. Parse args: --session, --index, --timeout, --plugin-root, --workspace-root from process.argv
    3. Derive workspaceRoot: from --workspace-root, or dirname of dirname of index path, or process.env.CLAUDE_PROJECT_DIR, or cwd
    4. Derive pluginRoot: from --plugin-root, or resolve the plugin directory relative to repl-sandbox.mjs location (using import.meta.url dirname parent -- repl-sandbox.mjs lives in plugins/lz-nx.rlm/scripts/, so parent is plugins/lz-nx.rlm/)
    5. Build options: if --timeout provided, pass as number; otherwise OMIT timeout from options so executeSandbox calls loadConfig for the default
    6. Call executeSandbox(code, options)
    7. Write JSON.stringify(result) to stdout
    8. Exit 0 (errors are in the result, not the exit code)

Create a temp session file for persistence tests.     Note: For tests that need the workspace index and git commands (files, search, nx globals), those are tested in repl-globals.test.mjs. The sandbox tests focus on VM execution, code transformation integration, session state, and the SandboxResult contract.
