  <behavior>
    code-transform.mjs - transformDeclarations(code):
    - "const x = 42;" -> wraps with Object.defineProperty(globalThis, "x", { value: (42), writable: false, enumerable: true, configurable: true })
    - "let y = 'hello';" -> "globalThis.y = 'hello';"
    - "var z = [];" -> "globalThis.z = [];"
    - Indented declarations (inside blocks) are NOT transformed (only column-0 matches)
    - Multiple declarations in same code block all transform correctly
    - Code with no declarations passes through unchanged
    - "const fn = () => { ... };" transforms correctly (single-line arrow)
    - "const obj = { a: 1, b: 2 };" transforms correctly (single-line object)
    - For const: the closing of Object.defineProperty must be handled. Strategy: find the statement terminator (semicolon at appropriate depth) and append ", writable: false, enumerable: true, configurable: true })". Use a post-processing pass after the regex replace to close open Object.defineProperty calls.
    - Edge case: "const x = 42" without trailing semicolon -- handle gracefully

    print-capture.mjs - createPrintCapture(maxPerCall=2000, maxTotal=20000):
    - Returns { print, getOutput, getTotalChars }
    - print("hello") captures "hello"
    - print(42) captures "42"
    - print(null) captures "null", print(undefined) captures "undefined"
    - print([1,2,3,4,5,6]) captures "Array(6) [1, 2, ... +4 more]"
    - print({large...}) truncates at 500 chars with "... [N chars]" suffix
    - print() with string > 2000 chars truncates with "... [N chars]"
    - After 20000 total chars, print() silently stops capturing
    - getOutput() joins all captured lines with "\n"
    - Circular reference in object falls back to String(value)
    - Multiple args: print("count:", 5) -> "count: 5"
  </behavior>
  <action>
    Create code-transform.mjs with a single exported function `transformDeclarations(code)`:
    1. Use regex `/^(const|let|var)\s+(\w+)\s*=/gm` to find top-level declarations
    2. For `let`/`var`: replace match with `globalThis.name =`
    3. For `const`: replace match with a marker, then in a second pass find the statement end and wrap with Object.defineProperty. Implementation approach: replace `const name =` with `Object.defineProperty(globalThis, "name", { value:` and then find the matching semicolon (tracking brace/bracket/paren depth) to append `, writable: false, enumerable: true, configurable: true });`. If no semicolon found (end of code), append the closing at end of code.
    4. Prepend `'use strict';` to the transformed code (required for const immutability enforcement per RESEARCH.md Pitfall 2).
    5. System prompt bans destructuring at top level so the regex is sufficient.

    Create print-capture.mjs with a single exported function `createPrintCapture(maxPerCall, maxTotal)`:
    1. Internal formatValue(value) function handles type-specific formatting per CONTEXT.md thresholds:
       - null/undefined -> String(value)
       - non-object -> String(value)
       - Array > 5 elements -> "Array(N) [first, second, ... +N-2 more]"
       - Object: try JSON.stringify(value, null, 2), truncate at 500 chars; catch -> String(value)
    2. print(...args) maps args through formatValue, joins with " ", enforces per-call (2000) and per-turn (20000) limits
    3. Silently stops capturing after maxTotal reached (no error thrown)

    Both modules: zero npm dependencies, ESM .mjs with node: prefix imports, JSDoc for all exports, single quotes, blank lines around control flow per CONVENTIONS.md.
  </action>
