<objective>
Harden repl-executor agent definition to prevent LLM deviation to `node -e` under cognitive load.

Purpose: UAT round 2, Test 2 (dependency query) failed because the LLM fell back to `node -e` with inline `fs.writeFileSync`/`fs.readFileSync` instead of using the Write tool + --file pattern. The fix adds explicit prohibitions in both `<role>` (highest influence) and `<execution>` sections, and updates structural tests to enforce their presence.

</objective>

- NEVER read or write the session file directly. Session state is managed transparently by the sandbox via the `--session` flag.
