      Navigates via the REPL fill/solve loop and returns only the
      distilled answer without intermediate exploration results.
    argument-hint: <question> [--debug]
    disable-model-invocation: true
    ---

    **Skill Body** -- workflow orchestration instructions for Claude:

    The skill body is a prompt that Claude (main conversation) follows step-by-step. It must:

    **Step 1: Validate input**
    - Check if `$ARGUMENTS` is empty or blank
    - If empty: respond with a usage hint message showing the invocation syntax and example questions. Do NOT spawn the agent. Stop here.
    - If non-empty: extract the question and detect `--debug` flag
    - Parse `--debug`: if `$ARGUMENTS` contains `--debug` (at start, end, or as a standalone token), set debug mode ON and strip the flag from the question text

    **Step 2: Determine paths**
    - Workspace root: use `git rev-parse --show-toplevel` via Bash (cross-platform, works in Git Bash)
    - Plugin root: derive from `${CLAUDE_SKILL_DIR}` by going up two levels (skill dir is `plugins/lz-nx.rlm/skills/explore/`, so `../../` is the plugin root). Use a Bash command: `dirname "$(dirname "${CLAUDE_SKILL_DIR}")"` -- but since `${CLAUDE_SKILL_DIR}` is not substituted in Bash commands either, instruct Claude to construct the path from the known skill location or use the workspace root + relative plugin path.
    - **Practical approach**: Instruct Claude to use `git rev-parse --show-toplevel` for workspace root. Then construct:
      - `INDEX_PATH`: `${WORKSPACE_ROOT}/tmp/lz-nx.rlm/workspace-index.json`
      - `PLUGIN_ROOT`: determined from the skill's known location in the installed plugin (Claude knows the plugin root from its plugin registry)
      - `SESSION_PATH`: `${WORKSPACE_ROOT}/.cache/repl-session-<id>.json` where `<id>` is a timestamp or UUID

    **Step 3: Ensure workspace index exists**
    - Check if the workspace index file exists at `INDEX_PATH`
    - If it does not exist: inform the user that the workspace index is being built, then run `node ${PLUGIN_ROOT}/scripts/workspace-indexer.mjs` to build it
    - If it exists: proceed

    **Step 4: Load config**
    - Read `maxIterations` from the config. Instruct Claude to read the config files in order (user override at `.claude/lz-nx.rlm.config.json`, then plugin default at `${PLUGIN_ROOT}/lz-nx.rlm.config.json`) and extract `maxIterations`. Default to 20 if neither file has it.
    - Also read `maxConsecutiveErrors` (default 3) and `maxStaleOutputs` (default 3)

    **Step 5: Generate session ID**
    - Create a unique session ID (timestamp-based, e.g., `Date.now()` formatted)
    - Construct `SESSION_PATH`: `${WORKSPACE_ROOT}/.cache/repl-session-${SESSION_ID}.json`

    **Step 6: Spawn the repl-executor agent**
    - Use the Task tool (or Agent tool -- Claude understands both names) to spawn the `repl-executor` agent
    - Construct the agent prompt containing:
      1. The user's question (with --debug stripped)
      2. `PLUGIN_ROOT=<absolute path>`
      3. `WORKSPACE_ROOT=<absolute path>`
      4. `INDEX_PATH=<absolute path>`
      5. `SESSION_PATH=<absolute path>`
      6. `MAX_ITERATIONS=<value from config>`
      7. `MAX_CONSECUTIVE_ERRORS=<value from config>`
      8. `MAX_STALE_OUTPUTS=<value from config>`
    - Set `max_turns` to `maxIterations + 2` as the external safety net

    **Step 7: Relay the result**
    - The agent returns plain text containing the FINAL answer
    - If the agent completed successfully: relay the answer to the user verbatim
    - If the agent did not complete (max_turns reached, no FINAL): inform the user "Exploration did not complete within the iteration limit."
    - If `--debug` was set: append a diagnostic footer with iteration count and timing information

    **Step 8: Clean up session**
    - If the agent completed successfully (returned a FINAL answer): delete the session file at `SESSION_PATH`
    - If the agent did not complete: keep the session file for post-mortem inspection, inform the user of its location

    Key constraints to embed in the skill instructions:
    - The skill runs in the MAIN conversation context (not forked)
    - The agent runs in an ISOLATED context -- this is the entire RLM thesis
    - Do NOT read intermediate agent results; only relay the final response
    - The skill does NOT run the REPL loop itself -- it delegates entirely to the agent
