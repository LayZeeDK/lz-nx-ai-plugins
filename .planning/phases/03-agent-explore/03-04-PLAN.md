<objective>
Close the two remaining UAT blockers that prevent the explore skill from running autonomously:

1. **GAP-01 (Permission prompts):** The temp-file + stdin redirect pattern from 03-03 still triggers 3 permission prompts per sandbox iteration (heredoc write to /tmp, stdin redirect `<`, out-of-tree delete). Root cause: `repl-sandbox.mjs` only accepts code via stdin (fd 0), forcing shell operators. Fix: add `--file` flag to sandbox, use Write tool for code creation, use in-tree `.cache/` path.

2. **GAP-02 (SKILL_DIR resolution):** The sequential dirname approach from 03-03 still fails because `${CLAUDE_SKILL_DIR}` is NOT expanded in skill markdown code blocks (known platform bug, GitHub #9354). The LLM types the literal `${CLAUDE_SKILL_DIR}` into Bash, triggering a `${}` permission prompt and getting `.` from dirname. Fix: derive PLUGIN_ROOT from WORKSPACE_ROOT (already resolved) + known relative path.

Purpose: These are the ONLY two remaining UAT failures. Fixing them unblocks v0.0.1 milestone closure.

Output: Modified `repl-sandbox.mjs` with `--file` flag, rewritten `repl-executor.md` with Write tool + `--file` invocation, rewritten `SKILL.md` Step 2 with WORKSPACE_ROOT derivation, updated structural tests.
</objective>
